# PROP-000: Base Project Rules {#root}

## Простыми словами {#plain-language}

Этот документ фиксирует базовые правила проекта: спеки являются каноном, код пишется после согласования спеки, а текущая работа ведется через `BOARD` и `WAL`.

## Goal {#goal}

Задать минимальные правила, чтобы NightLock Guard можно было реализовать маленькими проверяемыми шагами.

## Scope {#scope}

Входит:
- структура `specs/`;
- правило spec-first;
- адресация `spec://...#anchor`;
- разделение `PROP`, `FEAT` и `INFRA`;
- `@spec` markers для нового spec-owned кода.

Не входит:
- конкретное поведение ночной блокировки;
- выбор языка программирования;
- реализация Windows service.

## Rules {#rules}

- Человек задает канон через спеки.
- Реализация начинается только после согласования relevant `FEAT`/`INFRA`.
- `BOARD.md` показывает очередь и статус work items.
- `WAL.md` хранит короткое состояние активной работы.
- Новый или существенно измененный код должен ссылаться на governing specs через `@spec`.
- Осознанные ограничения, не закрытые текущей волной, фиксируются в `TECHDEBT.md`.

## Acceptance {#acceptance}

- В проекте есть self-contained spec-space.
- Каждый planned work item имеет governing spec.
- Дальнейшая реализация может ссылаться на стабильные `spec://` anchors.
