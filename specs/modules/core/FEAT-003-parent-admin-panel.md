# FEAT-003: Parent Admin Panel {#root}

## Простыми словами {#plain-language}

Родителю нужен простой экран настроек, чтобы не лезть в командную строку. В нем можно задать: во сколько включается ночная блокировка, во сколько выключается, на сколько минут пароль дает временный доступ, какая комбинация клавиш аварийно останавливает программу, и нужно ли блокировать клавишу Windows ночью. Этот экран защищен родительским паролем и не виден в обычном поиске Windows, чтобы ребенок его не нашел.

## Goal {#goal}

Дать родителю простой графический интерфейс для управления настройками NightLock Guard вместо CLI, оставаясь в границах безопасности продукта.

## Scope {#scope}

Входит:
- отдельное desktop-приложение настроек (parent admin app);
- доступ только после ввода родительского пароля;
- редактирование расписания (start/end), override-минут, hotkey-комбинации, тумблера блокировки Windows key;
- смена родительского пароля;
- сохранение настроек в общий машинный конфиг;
- скрытность от обычного поиска Windows (не security boundary, а удобство).

Не входит:
- удаленное управление / облако / синхронизация между ПК;
- разные расписания по дням недели (отдельный будущий FEAT);
- роли пользователей кроме одного родителя;
- восстановление забытого пароля;
- замена ACL-защиты конфигурации — панель пишет тот же защищенный конфиг.

## Access Control {#access-control}

- Перед показом любых настроек панель требует родительский пароль (`FEAT-002` verifier).
- Пароль вводится в маскированном поле и не логируется.
- Неуспешный ввод не открывает панель; после повторных неудач допускается короткая задержка.
- Панель не показывает текущий пароль и не хранит его в открытом виде.
- Запись изменений в конфиг разрешена только при пройденной аутентификации (и/или admin-правах, как требует `spec://modules/core/INFRA-001-windows-runtime-baseline#config`).

## Discoverability {#discoverability}

- Панель не создает ярлык в Start-меню и не индексируется обычным поиском Windows.
- Запускается из tray-меню helper и/или по известному пути установки.
- Скрытность — это слой удобства, а не граница безопасности; реальная защита — пароль и ACL (`spec://modules/core/PROP-001-product-boundaries#safety-boundaries`).
- Панель не маскируется под системный процесс и не прячет файлы в неожиданных местах (остается в обычном install path).

## Editable Settings {#editable-settings}

Панель может менять:
- `lock window start` (время включения, по умолчанию `23:30`);
- `lock window end` (время выключения, по умолчанию `08:00`);
- `override minutes` (1-240, по умолчанию 30) — на сколько пароль дает временный доступ;
- `emergency stop hotkey` — комбинация, останавливающая программу (`FEAT-004`);
- `Windows key suppression` — вкл/выкл блокировки Win key ночью (`FEAT-005`);
- родительский пароль (со старым паролем для подтверждения или admin-правами).

Правила валидации:
- start и end — корректное время суток; равные start/end запрещены (иначе окно пустое или круглосуточное);
- override minutes зажимается в 1-240;
- hotkey должен содержать минимум одну не-модификаторную клавишу и быть распознаваемым (`FEAT-004#hotkey-config`);
- невалидный ввод не сохраняется и показывает понятную ошибку, прежняя настройка остается.

## Persistence {#persistence}

- Сохранение пишет общий машинный конфиг (`spec://modules/core/INFRA-001-windows-runtime-baseline#config`), тот же, что читают service и helper.
- Изменения вступают в силу на следующем контролируемом reload конфига (старт сервиса/helper, явный refresh, или применение из панели).
- Панель не держит собственную скрытую копию настроек.

## UI Simplicity {#ui-simplicity}

- Один экран, понятные подписи на простом языке.
- Поля с дефолтами; кнопка Save и кнопка Cancel.
- Не тяжелый UI framework сверх необходимого (`spec://modules/core/INFRA-001-windows-runtime-baseline#resource-budget`); панель — короткоживущий процесс, запускается по требованию, не висит в фоне.

## Acceptance {#acceptance}

- Панель не открывает настройки без верного родительского пароля.
- Родитель может изменить start, end, override-минуты, hotkey и Win-key тумблер, и изменения применяются после reload.
- Невалидные значения отклоняются с понятной ошибкой и не портят конфиг.
- Панель не появляется в обычном поиске/Start-меню Windows, но доступна из tray helper или известного пути.
- Введенный пароль не логируется и не хранится в открытом виде.
- Панель пишет тот же ACL-защищенный конфиг и не создает скрытых дублей настроек.
