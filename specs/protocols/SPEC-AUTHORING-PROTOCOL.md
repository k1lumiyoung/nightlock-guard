# SPEC-AUTHORING-PROTOCOL

## Purpose

Use this protocol when creating or materially rewriting specs.

## Before Writing

Determine:
- whether the document is `PROP`, `FEAT`, or `INFRA`;
- whether it is a new spec, direct edit, or change spec;
- which specs govern it;
- what is in scope for the current wave;
- what is explicitly out of scope.

## Required Result

A good spec is:
- internally consistent;
- addressable through `spec://...#anchor`;
- specific enough for implementation;
- honest about open questions;
- scoped tightly enough that acceptance can be reached.

Do not hide uncertainty in vague wording. Narrow scope, add `REVIEW`, or ask the human.
